Getting Started
Login

Default admin credentials: admin / admin123 (change after first login).

Dashboard

The dashboard shows tool counts, error summaries, and provider group badges. Click any group badge to filter tools.

Explore Tools

The Tools page lists all 497 tools with group dropdown, description search, and clickable provider badges. Each tool has a schema viewer and execute interface.

Configuration

All config is in config/application.yml. API keys go in the fmp.api.key, fred.api.key, etc. sections. Override any key via SAJHA_ environment variables.

Tutorial: Creating an MCP Tool Two Methods
~15 minutes 2 files: JSON config + Python class
Step 1: Create config/tools/temperature_converter.json
{
  "name": "temperature_converter",
  "implementation": "sajha.tools.impl.temperature_converter.TemperatureConverterTool",
  "description": "Convert temperatures between Celsius, Fahrenheit, and Kelvin",
  "enabled": true,
  "metadata": {"category": "Utilities", "tags": ["temperature", "converter"]}
}
Step 2: Create sajha/tools/impl/temperature_converter.py
from sajha.tools.base_mcp_tool import BaseMCPTool

class TemperatureConverterTool(BaseMCPTool):
    def __init__(self, config=None):
        super().__init__(config or {'name': 'temperature_converter',
                                     'description': 'Convert temperatures'})

    def get_input_schema(self):
        return {"type": "object", "properties": {
            "value": {"type": "number"},
            "from_unit": {"type": "string", "enum": ["celsius", "fahrenheit", "kelvin"]},
            "to_unit": {"type": "string", "enum": ["celsius", "fahrenheit", "kelvin"]}
        }, "required": ["value", "from_unit", "to_unit"]}

    def get_output_schema(self):
        return {"type": "object", "properties": {
            "converted_value": {"type": "number"}, "converted_unit": {"type": "string"}}}

    def execute(self, arguments):
        v, f, t = arguments['value'], arguments['from_unit'], arguments['to_unit']
        c = v if f == 'celsius' else (v - 32) * 5/9 if f == 'fahrenheit' else v - 273.15
        r = c if t == 'celsius' else c * 9/5 + 32 if t == 'fahrenheit' else c + 273.15
        return {"converted_value": round(r, 2), "converted_unit": t}
Step 3: Deploy

The hot-reload system picks up new tools automatically. Or restart: python run_server.py

curl -X POST http://localhost:3002/api/tools/execute \
  -H "X-API-Key: YOUR_KEY" -H "Content-Type: application/json" \
  -d '{"tool": "temperature_converter", "arguments": {"value": 100, "from_unit": "celsius", "to_unit": "fahrenheit"}}'
# → {"converted_value": 212.0, "converted_unit": "fahrenheit"}

from sajha.studio import sajhamcptool

@sajhamcptool(description="Convert temperatures", category="Utilities")
def convert_temperature(value: float, from_unit: str, to_unit: str) -> dict:
    c = value if from_unit == 'celsius' else (value - 32) * 5/9 if from_unit == 'fahrenheit' else value - 273.15
    r = c if to_unit == 'celsius' else c * 9/5 + 32 if to_unit == 'fahrenheit' else c + 273.15
    return {"converted_value": round(r, 2), "converted_unit": to_unit}

API Quick Reference
3 auth methods: X-API-Key header (recommended for automation), Authorization: Bearer JWT, or sajha_token session cookie. Full reference: /docs/view/API_Reference.md
API Key Authentication
# Execute a tool
curl -X POST http://localhost:3002/api/tools/execute \
  -H "X-API-Key: sk_live_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "tool": "yahoo_finance_quote",
    "arguments": {"symbol": "AAPL"}
  }'
JWT Authentication
# Get token
curl -X POST http://localhost:3002/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"user_id": "admin", "password": "admin123"}'

# Use token
curl -X POST http://localhost:3002/mcp \
  -H "Authorization: Bearer eyJ..." \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":"1","method":"tools/list"}'
Python SDK
from sajhaclient import (
    SajhaConfig, SajhaClient,
    MCPClient, ApiKeyAuth
)

config = SajhaConfig(
    base_url="http://localhost:3002"
)
auth = ApiKeyAuth("sk_live_abc123")

# REST client
client = SajhaClient(config, auth=auth)
tools = client.list_tools()
result = client.execute_tool(
    "fred_gdp",
    {"start_date": "2024-01-01"}
)

# MCP client (JSON-RPC 2.0)
mcp = MCPClient(config, auth=auth)
mcp.initialize()
mcp_tools = mcp.list_tools()
mcp_result = mcp.call_tool(
    "yahoo_finance_quote",
    {"symbol": "TSLA"}
)
MCP Protocol (JSON-RPC 2.0)
# MCP initialize
curl -X POST http://localhost:3002/mcp \
  -H "X-API-Key: sk_live_abc123" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":"1","method":"initialize","params":{"protocolVersion":"2025-06-18"}}'

# MCP tools/call
curl -X POST http://localhost:3002/mcp \
  -H "X-API-Key: sk_live_abc123" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":"2","method":"tools/call","params":{"name":"calc_compound_interest","arguments":{"principal":10000,"rate":0.05,"years":10}}}'

MCP Studio — Visual Tool Creation

MCP Studio provides 7 tool creator types. Each generates both a JSON config and Python implementation, deployed instantly via hot-reload.

Python Code
Write @sajhamcptool decorated functions. Type hints become JSON Schema automatically.
REST Service
Wrap any REST API as an MCP tool. Supports GET/POST/PUT/DELETE, JSON/CSV/XML responses, API Key and Basic Auth.
DB Query
Parameterized SQL templates against DuckDB, SQLite, PostgreSQL, or MySQL.
Script
Execute Bash, Python, Node.js, Ruby, or Perl scripts with array-of-string arguments.
PowerBI Report
Export PowerBI reports as PDF/PPTX/PNG via Azure AD Service Principal.
PowerBI DAX
Execute EVALUATE DAX queries against PowerBI datasets with parameterized inputs.
LiveLink (OpenText Content Server)
Search, list, get metadata, and download documents from OpenText ECM. Supports Basic Auth, OAuth, and OTDS.
Tip: Use {param} in REST URLs, {{ '{{param}}' }} in SQL, @param in DAX.

Troubleshooting

  1. Verify the user exists in the database (or config/users.json for legacy import)
  2. Check that the account is enabled
  3. Clear browser cookies and retry
  4. Check server console for auth errors

  1. Check that required API keys are set in config/application.yml (e.g., fmp.api.key, fred.api.key)
  2. Verify the tool is enabled: GET /api/tools/list
  3. Check the tool's input schema: GET /api/tools/{name}/schema
  4. Review server logs for stack traces

  1. HTTP POST transport: POST http://localhost:3002/mcp
  2. SSE transport: GET /mcp/sse for event stream, POST /mcp/message for messages
  3. Verify auth token/API key is valid
  4. Check /health endpoint for server status

Page Glossary

MCP: Model Context Protocol — a standardized interface for AI tool discovery and execution via JSON-RPC 2.0.

JSON-RPC 2.0: The protocol encoding used for MCP request/response communication.

SSE: Server-Sent Events — one of two MCP transports, alongside HTTP POST.

A2A: Agent-to-Agent protocol for multi-agent orchestration.

Hot-Reload: Automatic config reload without server restart.

API Key: An X-API-Key credential with tool-level permissions.
