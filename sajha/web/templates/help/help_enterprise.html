Three MCP Transports
HTTP POST /mcp

Stateless. One request = one response. Best for automation scripts, CI/CD pipelines, simple tool calls.

SSE /mcp/sse

Server-push streaming. Persistent connection for long-running tools, progress updates, web UI real-time feeds.

WebSocket /mcp/ws

Full-duplex bidirectional. Server pushes notifications without polling. Best for interactive agents. Connect: ws://host:3002/mcp/ws?token=jwt

OpenTelemetry Observability
Metrics

Per-tool p50/p95/p99 latency. GET /api/metrics/tools for all tools, /api/metrics/tools/{name} for one.

Alerts

Error spike (>10/5min), high latency (p95 >5s). Custom rules with webhook support.

Health Probes

/health (liveness) and /ready (readiness). Kubernetes-ready.

OTEL Export

Optional OpenTelemetry SDK exports traces and metrics to Datadog, Grafana, Splunk, Jaeger.

Multi-Tenancy

Tenant-isolated tool access with wildcard patterns + blocked lists. Per-tenant quotas (daily/monthly calls, LLM tokens). Data isolation via directory prefixes.

API: GET/POST/PUT/DELETE /api/tenants

Plugin System

Drop a plugin directory in config/plugins/ with plugin.json manifest + tool configs. Auto-discovered, validated (SHA-256 checksum), dependencies installed.

API: GET /api/plugins, POST /api/plugins/{name}/load

Tool Versioning

Run v1 and v2 side-by-side. Deprecation lifecycle: active → deprecated → sunset → retired. Contract testing validates all tools.

API: POST /api/contract-test runs all {{ tool_stats.total_tools }} tools.

Operations (v5.2.0)
Tool Caching

Per-tool output cache with configurable TTL via cache_ttl in tool JSON config. LRU eviction, automatic expiry. API: GET /api/cache/stats

Circuit Breakers

Per-provider failure tracking. CLOSED → OPEN (5 failures) → HALF_OPEN (60s recovery probe). API: GET /api/circuits

Webhooks

Event-driven notifications: tool.completed, task.failed, circuit.opened. 3 retries with backoff. API: POST /api/webhooks/subscribe

Shell Tools

Sandboxed Python + Bash. Allowlisted imports/commands, resource limits, full audit trail. Disabled by default. shell.enabled: true

Async Execution

Background tool execution with delivery routing. Worker pool, bounded queue, webhook/Kafka/file delivery. API: POST /api/tools/{name}/execute-async

Audit Log

Structured security events to DB. 11 event types. Queryable: GET /api/audit?action=login_failed
