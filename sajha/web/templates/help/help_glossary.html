Glossary

Back to Help
MCP — Model Context Protocol

Open standard for AI tool integration. JSON-RPC 2.0 over HTTP, SSE, or WebSocket. Defines methods for tools/list, tools/call, resources, prompts, and completions.

JSON-RPC 2.0

Wire protocol for MCP. Every request: {"jsonrpc":"2.0", "method":"tools/call", "params":{...}, "id":1}. Response: {"result":{...}} or {"error":{...}}.

Tool

A callable function exposed via MCP. Has a name, description, inputSchema (JSON Schema), and execute method. SAJHA has {{ tool_stats.total_tools }} built-in tools.

Composite Tool

A tool that orchestrates multiple tools in one call. Sibling (parallel) or Parent-Child (fan-out). Defined in DB, schemas auto-generated.

StepResult — Kleisli Envelope

Monadic wrapper for tool execution results. Carries value, error, trace, duration, and confidence. Errors short-circuit the pipeline.

ParamLens

A view/set pair that projects parent output into child params. Syntax: $.field (from record), $input.field (from original input), literals.

EntropyGuard

Tracks cumulative confidence through a composite pipeline. Sequential = multiply. Parallel = min (weakest link). Can refuse execution if entropy exceeds threshold.

Confidence Score

Float 0.0–1.0 indicating tool reliability. 1.0 = deterministic (calculator), 0.95 = stable API (FRED), 0.80 = web crawl. Composites compound scores.

TransportCoalgebra

Abstract interface for MCP transport clients. step(method, params) → (result, new_state). HTTPTransport, SSETransport, WSTransport are bisimilar implementations.

Bisimulation

Behavioral equivalence test. If two transports produce the same outputs for the same inputs, they're bisimilar — safe to hot-swap at runtime.

ClientPipeline

Client-side tool composition. Chains tools where output feeds the next. Same $. / $input. param mapping as server composites. Tracks confidence.

MCPHandler

Core server component. Routes all 12 MCP methods. Shared across HTTP POST, SSE, and WebSocket transports. Handles auth, tool execution, audit logging.

ToolsRegistry

Singleton holding all registered tools. Loaded from JSON configs at boot. Supports hot-reload. Methods: register_tool(), get_tool(), get_all_tools().

LLM Gateway

Multi-provider AI inference layer. 6 providers (Anthropic, OpenAI, Bedrock, Together, Ollama, Azure). Registry-based factory. DB-managed models.

MCP Studio

Visual tool creation platform. 9 creator types: Python, REST, DB Query, Script, PowerBI, DAX, LiveLink, SharePoint, OLAP. Zero-code tool building.

application.yml

Single configuration file driving everything: version, paths, DB connection, logging, plugins directory. Supports ${ENV_VAR:default} substitution.
