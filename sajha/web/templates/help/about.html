About SAJHA MCP Server

A Production-Grade Model Context Protocol Implementation

Overview

SAJHA MCP Server is a comprehensive Python implementation of the Model Context Protocol (MCP) built on FastAPI. It serves 497 tools across financial markets, government data, search engines, analytics, and enterprise integrations through a single MCP-compliant endpoint.

The server supports three transports (HTTP POST, SSE, WebSocket), an embedded LLM gateway with 6 providers, semantic tool discovery, composite tool orchestration, OpenTelemetry, multi-tenancy, and a plugin system. Version {{ app_version }} is fully compliant with MCP 2025-11-25 — Tasks, Elicitation, Sampling with tool calling, tool icons, and Origin validation for Streamable HTTP.

The Meaning of SAJHA
साझा

SAJHA (Hindi: साझा, Urdu: ساجھا) means "shared," "common," "mutual," or "collaborative."

True to its name, SAJHA is a shared bridge between AI systems and the world's data — democratizing access to hundreds of tools through one standardized protocol, with collaborative tool creation via MCP Studio.

MCP Specification Compliance
Protocol Version: 2025-11-25 (latest)
Released November 25, 2025 by the MCP Steering Group
Fully Compliant
Tasks (SEP-1686)

Async tracking for long-running requests. Create, poll, cancel. States: working → completed/failed/cancelled.

Elicitation (SEP-1330)

Server-initiated user input. Form mode (JSON Schema forms) and URL mode (OAuth/consent redirects).

Sampling + Tools (SEP-1577)

Server-initiated LLM calls with tool definitions and toolChoice. Enables server-side agent loops.

Icons (SEP-973)

Icon metadata for tools, resources, and prompts. URL and emoji icon types for richer UI rendering.

TRANSPORT COMPLIANCE

HTTP POST /mcp (stateless) · SSE /mcp/sse (Streamable HTTP with Origin validation) · WebSocket /mcp/ws (full-duplex). JSON-RPC 2.0 on all three. JSON Schema 2020-12 dialect.

SECURITY COMPLIANCE

Tool errors return isError:true (not protocol errors) for model self-correction. Origin header validation (HTTP 403). PKCE. OAuth 2.1. RBAC + API key + JWT + OAuth SSO.
