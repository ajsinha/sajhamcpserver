Getting Started
1. Browse Tools

Go to Tools → search or filter by category → click any tool to see its schema, parameters, and description.

2. Execute a Tool

Click Execute on any tool → fill the parameter form → click Run. Results appear with a human-readable summary + raw JSON.

3. Connect via API

Create an API key at Admin → API Keys. Use it with X-API-Key header or the Python SDK: SajhaClient(config, auth=ApiKeyAuth("key"))

Create a Tool with MCP Studio
  1. Go to MCP Studio and choose a creator type (Python, REST, DB Query, Script, etc.).
  2. Fill in the tool name, description, and parameters. JSON Schema is auto-generated.
  3. Add the data source — a Python function, REST URL, SQL query, or script file.
  4. Click Test to run with sample arguments and verify the output.
  5. Click Save. The tool config is written to config/tools/ and hot-reloaded within seconds.
  6. Your tool is now available via tools/list and tools/call on all three transports.
Build a Composite Tool
  1. Go to MCP Studio → Composite Builder.
  2. Enter a name (e.g., market_snapshot) and choose arrangement: Sibling (parallel) or Parent-Child (fan-out).
  3. Set the Master tool — this runs first and its output feeds the steps.
  4. Click + Add Step to add child tools. Set output key, param mapping ($.field), and static params.
  5. Watch the Flow Diagram update live as you add steps. Drag steps to reorder.
  6. Click Preview Schema to see auto-generated input/output schemas.
  7. Click Save. The composite is registered and available via MCP immediately.
Connect an LLM Provider
  1. Go to AI → LLM → Providers tab.
  2. Click Add Provider. Choose type: Anthropic, OpenAI, Bedrock, Together, Ollama, Azure, or Custom.
  3. Enter the required credentials (API key, endpoint, region, etc.).
  4. Save the provider, then switch to the Models tab.
  5. Click Add Model, select the provider, enter model ID (e.g., claude-sonnet-4-20250514).
  6. Set your preferences in the Preferences tab — default provider, model, temperature.
Create a Plugin
  1. Create a directory: config/plugins/my-plugin/
  2. Add plugin.json with name, version, description, and tool list.
  3. Add tool JSON configs in a tools/ subdirectory, or Python tool classes.
  4. Call POST /api/plugins/discover to scan for new plugins.
  5. Call POST /api/plugins/my-plugin/load to install dependencies and register tools.
  6. Your plugin's tools are now available via MCP tools/list and tools/call.
Set Up Multi-Tenancy
  1. Create a tenant: POST /api/tenants with name, tool_patterns, and quotas.
  2. Assign users to the tenant via the Admin panel or API.
  3. Set tool access patterns: ["fred_*", "calc_*"] — wildcards with fnmatch.
  4. Set quotas: max calls per day/month, max LLM tokens.
  5. Tenant users can only access tools matching their patterns. Quotas auto-reset.
Connect the Python SDK
  1. Install: pip install sajhaclient
  2. from sajhaclient import SajhaClient, SajhaConfig, ApiKeyAuth
  3. client = SajhaClient(SajhaConfig(base_url="http://host:3002"), auth=ApiKeyAuth("sja_key"))
  4. result = client.execute_tool("yahoo_quote", symbol="AAPL")
  5. For pipelines: ClientPipeline(client).add_step(...).execute({...})
  6. For WebSocket: MCPWebSocketClient(config, auth) with .connect() + .call_tool()
