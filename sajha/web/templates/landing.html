Capabilities

Not a tool server. An intelligence server.

SAJHA embeds the LLM gateway, so it can reason about tools, resolve intent, compose workflows, and track costs. Other MCP servers are dumb pipes.

Embedded LLM Gateway

Anthropic, OpenAI, Bedrock, Together, Ollama, Azure — all via official SDKs. Registry factory for custom providers. DB-managed models with user-level overrides.

Semantic Tool Discovery

Describe what you need in English. Vector embeddings of 497 tool descriptions resolve intent to the right tool with confidence scores and auto-extracted parameters.

Composite Tools

Chain multiple tools into one call. Sibling pattern runs tools in parallel. Parent-Child fans out per record. Dynamic schemas built at boot from DB definitions.

OpenTelemetry Observability

Per-tool p50/p95/p99 latency histograms. Error spike alerting. Health probes for Kubernetes. Optional OTEL SDK exports to Datadog, Grafana, Splunk.

Multi-Tenancy

Tenant-isolated tool access with wildcard patterns. Per-tenant quotas, data isolation, and API key pools. Enterprise-ready from day one.

Plugin System

Drop a plugin directory with JSON tool configs or Python classes. Auto-discovered, checksum-verified, dependency-installed. Publish and share tool packs.

Multi-Layer Security

Four auth methods: JWT, API Key, Session Cookie, OAuth SSO. RBAC with wildcard tool permissions. Every call authenticated, authorised, and audit-logged.

MCP Studio

7 visual tool creators: Python, REST, DB Query, Script, PowerBI Report, PowerBI DAX, LiveLink. Analysts create tools without writing code. Hot-reload deploys in seconds.

Tool Versioning

Run v1 and v2 side-by-side. Deprecation lifecycle with sunset dates. Contract testing validates all 497 tools against live APIs. Zero surprises in production.

Connectivity

Three ways to connect. One protocol.

Every transport uses the same MCPHandler — identical JSON-RPC 2.0, same 12 methods, same auth, same audit logging.

POST /mcp

HTTP POST

Stateless request-response. One call, one result. Best for automation scripts, CI/CD pipelines, and simple tool execution.

GET /mcp/sse

Server-Sent Events

Server-push streaming. Persistent connection for long-running tools, progress updates, and real-time web UI feeds.

WS /mcp/ws

WebSocket

Full-duplex bidirectional. Server pushes notifications without polling. Best for interactive agents and real-time dashboards.
